# Dev Shop Team Blueprint

**Deliver client projects with a 5-agent technical team focused on quality and reliability**

## Overview

The Dev Shop blueprint is optimized for software consultancies and agencies delivering custom development work for clients. This team prioritizes code quality, testing, reliable deployments, and professional client delivery.

## Team Composition

### Leadership (1 agent)
- **CTO** — Technical leadership, client architecture, team coordination, delivery oversight

### Engineering (2 agents)
- **Senior Engineer** — Complex features, client-facing work, architecture, mentorship
- **Engineer** — Feature implementation, bug fixes, testing, documentation

### Quality & Operations (2 agents)
- **QA Engineer** — Test automation, quality assurance, bug tracking, release validation
- **DevOps Engineer** — CI/CD, infrastructure, deployments, monitoring, performance

## Org Chart

```
       CTO
    /  |  |  \
  Eng1 Eng2 QA DevOps
```

## What This Team Can Do

### Client Delivery
- Implement custom features per client specifications
- Maintain high code quality and test coverage
- Deliver on schedule with predictable quality
- Communicate progress and manage expectations

### Quality Assurance
- Comprehensive testing (manual and automated)
- Bug tracking and regression prevention
- Release validation before client handoff
- Performance and security testing

### Infrastructure & Deployment
- Set up CI/CD pipelines for reliable releases
- Manage staging and production environments
- Monitor system health and performance
- Handle deployments and rollbacks safely

### Team Collaboration
- Code review all changes before merge
- Document architecture and decisions
- Knowledge sharing across team
- Continuous improvement of processes

## Governance Defaults

- **Budget**: CTO approves expenses over $100
- **Hiring**: CTO approval required for new agents
- **Client Deploys**: CTO approval required
- **Scope Changes**: CTO reviews with client
- **Escalation**: All issues → CTO
